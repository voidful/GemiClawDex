附錄 A

AI 深度分析 Prompt

給 GPT / Codex / Claude / Gemini 的長版分析任務書，而不是只有幾句泛用提示詞

這份 prompt 的核心差異

它強制模型做三件事：第一，對照實際原始碼判斷功能狀態；第二，把你列出的每一份參考資料都納入 coverage matrix；第三，直接產出文件網站的 IA、章節規劃與設計 brief。

它會要求模型輸出什麼

A

Codebase 實況檢查

區分「已完成」、「部分實作」與「研究方向」，避免把基本架構誤寫成已上線功能。

B

Reference Coverage Matrix

每一份參考來源都要對應出「提取到的訊號、對 GCD 的借鏡點、建議實際做出來位置」。

C

Borrow / Build / Avoid

不是只比較功能，而是整理成可執行的工程決策矩陣。

D

文件網站設計 Brief

同步產出章節順序、讀者分層、視覺方向與動態互動建議。

Prompt 片段預覽

ANALYSIS_PROMPT.md 
你現在是「AI AI 助手 首席架構分析師 + 技術總編輯 + 文件網站策展人」。

硬性規則：
1. 不得把「有檔案 / 有型別 / 有註解」直接視為「功能已完整上線」。
2. 每個重要判斷都要標註來源：原始碼路徑或外部參考連結。
3. 必須把功能狀態分成：
   - Implemented（已完成）
   - Partially Wired（部分實作）
   - Stub / Planned（佔位 / 規劃中）
   - Inferred（推測功能）
4. 必須完整覆蓋所有參考資料；如果連結失效，也要在 Reference Coverage Matrix 保留該列。
5. 最後要產出文件網站 IA、章節規劃與設計 brief。

建議搭配的輸入

  • README.md 與 docs/architecture.md
  • crates/gcd-core/src/app.rs、agent.rs、prompt.rs、providers.rs、session.rs
  • plugins.rs、mcp.rs、tools/registry.rs 等擴充點
  • 整個 docs/ 目錄，讓模型能同時比對「現有文件」與「實際實作」
完整檔案：docs/ANALYSIS_PROMPT.md
