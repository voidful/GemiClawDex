Context Engineering — 你送進去的東西決定了 AI 的表現

Prompt Engineering 只是 Context Engineering 的子集。這一章解釋三層演進，以及 GCD 怎麼組裝 context。

一句話解釋

Context Engineering 是「決定送什麼資訊給 AI 模型」的工程實踐。Prompt 只是 context 的一小部分。真正影響 AI 表現的，是你送進去的所有東西：檔案內容、工具結果、記憶、專案規範、技能描述。

三層演進：從 Prompt 到 Harness

很多人把「跟 AI 對話」和「寫 prompt」畫等號。但實際上，AI 助手收到的不只是你打的那句話。中間有一整條管線在組裝 context。理解這三層，是理解 harness engineering 的前提。

層次 你在做什麼 範例 GCD 對應
Prompt Engineering 決定「怎麼問」 「請用 bullet point 列出這段程式碼的問題」 使用者在 REPL 輸入的文字
Context Engineering 決定「送什麼進去」 把相關檔案、測試結果、專案規範一起送給模型 prompt.rs 的組裝邏輯
Harness Engineering 決定「整個系統怎麼運作」 工具呼叫迴圈、權限控制、session 管理、驗證迴圈 agent.rs + 所有基礎設施

換個比喻：如果 AI 模型是一個很厲害的廚師，Prompt Engineering 是告訴他「做一道義大利麵」。Context Engineering 是把正確的食材、食譜、廚房工具都準備好放在他面前。Harness Engineering 是設計整個廚房的動線、安全規範、品質檢查流程。

GCD 的 Context 從哪裡來

當你在 GCD 輸入一句話，模型實際收到的 context 遠比那句話多。以下是 prompt.rs 組裝的完整結構：

1
System Prompt 基礎指令 — 告訴模型它是一個 coding agent，有哪些工具可用，行為規範是什麼。
2
AGENTS.md 專案規範 — 從 instructions.rs 載入。也支援 GEMINI.md、CLAUDE.md、GCD.md。告訴模型這個專案的慣例。
3
Memory 注入 — MEMORY.md（環境知識）和 USER.md（使用者偏好）的內容，由 prompt.rs 的 memory context injection 處理。
4
Skill Catalog 列表 — 可用技能的 metadata（名稱 + 描述），不是完整內容。這就是 Progressive Skill Disclosure 的第一層。
5
使用者的提示詞 — 你實際輸入的文字，包括 @{file} 檔案注入和 !{command} shell 區塊。
6
對話歷史 — 之前的 user/assistant 訊息對。如果太長，agent.rs 會自動壓縮（auto-compact）。
7
工具呼叫結果 — 上一輪工具執行的輸出（例如 read_file 的內容、shell 的 stdout）。
原始碼：crates/gcd-core/src/prompt.rs、crates/gcd-core/src/prompt/expansion.rs

Context Window 是有限的

所有模型都有 context window 上限（例如 128K tokens）。這意味著上面七層的內容加起來不能超過這個上限。如果超過了，GCD 的 agent.rs 會自動壓縮對話歷史。

這就是為什麼 Context Engineering 很重要：你需要決定什麼資訊值得佔用有限的 context 空間。Anthropic 把這個概念叫做「Context Anxiety」，意思是模型的注意力是有限的，塞太多東西進去反而會讓它分心。

和 Hermes Agent 的對照

Hermes Agent 的 context_compressor.py（37K 行）和 context_engine.py 處理同樣的問題，但規模大得多。GCD 的壓縮邏輯在 agent.rs 的 auto-compact 區段，相對簡潔，更適合學習核心概念。

動手練習

試試看用 GCD 執行同一個任務，但送進去的 context 不同：

實驗做法觀察重點
A：只有 prompt gcd exec "這個專案用了什麼設計模式" 模型只能猜測，回答很泛
B：加上檔案注入 gcd exec "分析 @{crates/gcd-core/src/agent.rs} 用了什麼設計模式" 模型能引用具體的程式碼
C：加上 AGENTS.md 確認 AGENTS.md 存在後重跑 B 模型知道設計決策的來源和理由

差異就是 Context Engineering 的效果。
