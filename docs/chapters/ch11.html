技能系統 (Skills)

可選注入的專業知識 — skills.rs

技能 vs 命令

維度命令 (Commands)技能 (Skills)
格式TOMLMarkdown (SKILL.md)
觸發方式主動呼叫 (/command)選擇注入
注入位置替換使用者輸入追加到 System Prompt
用途定義任務模板提供專業知識上下文

技能目錄結構

技能可以是單一 .md 檔案，或包含 SKILL.md 的目錄：

技能目錄結構 
.gcd/skills/
├── rust-patterns.md           ← 單檔技能
├── security-audit/
│   └── SKILL.md               ← 目錄技能
└── api-design/
    └── SKILL.md               ← 目錄技能

技能的第一個非空行會被擷取為摘要描述，供選擇時顯示。如果 SKILL.md 包含 YAML frontmatter（以 --- 分隔的區塊），則會解析 name、description、version、platforms 欄位。完整內容在選中後注入到 prompt 中。

YAML Frontmatter（可選）

SKILL.md 可以在開頭加入 YAML-like 的 frontmatter，提供結構化的 metadata：

範例：含 frontmatter 的 SKILL.md 
---
name: rust-patterns
description: Idiomatic Rust patterns for error handling and lifetimes
version: 1.2.0
platforms: [macos, linux]
---

# Rust Patterns

When working with Rust code, prefer...

platforms 欄位可用於過濾：如果當前平台不在清單中，該 skill 不會被載入。沒有 platforms 欄位的 skill 在所有平台都可用。

三層漸進載入（Progressive Disclosure）

為了節省 token，技能系統支援三個層次的載入：

層次內容何時使用
Tier 1只顯示 (name, description) 清單在系統 prompt 中列出所有可用 skill，讓 agent 知道有哪些技能
Tier 2完整 SKILL.md 內容 + version使用者透過 /skill <name> 選中時
Tier 3linked files 清單（references/, templates/, assets/）選中時一併列出，agent 可用 read_file 按需載入

Tier 1 由 SkillCatalog::metadata_listing() 提供，只佔很少 token。Tier 2+3 只在 skill 被明確選中後才注入 prompt。

skill_manager 工具：runtime 技能 CRUD

參考 hermes-agent 的學習迴圈設計，skill_manager 讓 agent 在成功完成任務後，可以自行呼叫此工具將做法固化為可重用的 skill。支援六個 action：

Action作用
create建立新 skill（name + SKILL.md 內容）
edit完整替換 SKILL.md 內容
patchfind-and-replace，目標文字必須唯一
delete刪除整個 skill 目錄
write_file新增或覆寫 skill 目錄下的支援檔案
remove_file刪除 skill 目錄下的支援檔案

只能修改 user-created skills（~/.gcd/skills/），bundled 與project-level skills 為唯讀。所有寫入前會執行安全掃描：prompt injection pattern、exfiltration pattern、invisible unicode characters。

原始碼：crates/gcd-core/src/skills.rs、crates/gcd-core/src/tools/skill_manager.rs
