這個網站的重點

我們不是要推銷某個 AI 產品有多厲害。重點是讓你理解：一個 AI 程式助手需要哪些零件？這些零件怎麼接在一起？為什麼這樣設計？視覺和動畫只是幫你建立直覺，真正的內容都有程式碼可以對照。

這個網站幫你回答的四個問題

1

AI 助手的「外殼」是什麼？

AI 模型只會生成文字。要讓它變成能讀檔案、跑指令、修改程式碼的助手，需要在外面包一層工程系統（提示詞組裝、工具管理、安全範圍、工作記錄⋯）。

2

GCD 做到了哪些功能？

每個功能都有實際的 Rust 程式碼，並且通過自動化測試。不是說說而已。

3

和其他產品比，差在哪？

不只是比功能多少，而是告訴你每個設計背後的取捨：學了什麼、沒學什麼、為什麼。

4

我可以怎麼用它來學習？

提供閱讀路線、程式碼入口點和練習方向。讓你從「讀文件」走到「讀原始碼」再走到「自己動手改」。

trust、permission、sandbox 這些名詞搞不清楚 名詞速查 8 個最容易搞混的概念，用白話文一次講清楚。
同一個任務換不同 AI 模型（GPT、Gemini、Claude）會怎樣？ Provider 章 不同模型的 API 格式完全不一樣，GCD 怎麼讓你一鍵切換。
AI 跑太久、記憶體爆掉、API 掛了怎麼辦？ Runtime 章 自動壓縮對話、備援切換、token 用量追蹤⋯⋯ 這些保護機制怎麼運作。
權限、沙盒、提示詞怎麼一起配合？ 流程章 從你打字到 AI 真的改了檔案，中間每一步驟的完整流程。
工作記錄（session）可以回放嗎？ 執行證據章 每次執行都會留下完整記錄，可以回放、續接、分叉。
container 沙盒到底隔離了什麼？ Container 章 哪些指令會在 Docker 容器裡跑、哪些還是在你的電腦上跑。
MCP、plugin 和內建工具差在哪？ 擴充章 三種不同層次的工具擴充方式，各自的安全範圍和限制。
hooks 是什麼？可以在 AI 執行前後做事嗎？ 政策章 怎麼在 AI 呼叫工具前後插入自定義檢查或動作。
trust 規則具體影響了哪些功能？ Trust 章 哪些專案設定只有在信任的工作區才會載入、哪些一律可用。
最終送進 AI 的提示詞是怎麼拼出來的？ Prompt 章 你的輸入、專案規則、工具清單、歷史記錄是怎麼被組合成一份完整提示詞的。
AGENTS.md、command、skill 分別要怎麼用？ 工作流章 專案規範、任務模板、方法論能力包：三種不同的設定方式和適用時機。
